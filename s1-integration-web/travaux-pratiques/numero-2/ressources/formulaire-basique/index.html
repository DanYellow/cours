Le but de cette pratique est de construire notre premier formulaire en y ajoutant au fur et à mesure des éléments tels que la balise <label /> ou encore <select>. Les formulaires sont des éléments clés du web 2.0 et 3.0, grâce à eux, et aux serveurs, l'utilisateur peut communiquer avec le site ou même d'autres internautes. Ainsi, il est primordial d'apporter beaucoup de soin à la construction des formulaires pour éviter de nuire à l'expérience de vos internautes. Sur un bon site, un internaute prend toujours du plaisir à y revenir.

A noter que la gestion des formulaires va de pair avec le cours "R1.12 - Développement Web" où vous apprendrez à gérer les données en provenance d'un formulaire.

Quand vous travaillez avec VSCode. Pensez toujours à glisser-déposer votre dossier de travail dans VS Code. Ceci vous permettra d'accéder très facilement à tous les fichiers de votre projet.

Formulaire basique — Consignes TP

Partie 1 (cliquer pour afficher / cacher)

Sauf indication contraire, l'ensemble des champs crées doivent être dans un seul et unique formulaire (balise <form>).

  1. Créez une nouvelle page html avec un formulaire (balise <form>)
    • Voir documentation de la balise <form> (ouverture dans un nouvel onglet)
  2. Ajoutez l'attribut "action" avec la valeur "" pour la balise <form>
  3. Ajoutez une balise <input> de type "text"
    • Ne pas oublier de rendre le champ accessible
      • Mettre une balise <label>
        • Voir documentation de la balise <label> (ouverture dans un nouvel onglet)
      • Associer le champ à sa balise label grâce aux attributs "for" et "id"
    • Voir documentation de la balise <input type="text"> (ouverture dans un nouvel onglet)
  4. Ajoutez une autre balise <input> de type "text" avec une valeur par défaut en utilisant l'attribut "value"
  5. Ajoutez dans le même formulaire une autre balise <input> d'un type different de "text", vous pourrez trouver la liste des types dans la documentation. Tout en prenant soin de rendre le champ accessible
    • Voir documentation des types possibles pour la balise <input> (ouverture dans un nouvel onglet)

Partie 2 (cliquer pour afficher / cacher)

  1. Rajoutez l'attribut "method" avec la valeur "GET" au formulaire

    Même si la valeur "GET" est la valeur par défaut de l'attribut "method", il est préférable de toujours préciser la valeur de l'attribut "GET".

  2. Rajoutez un élément de formulaire permettant sa soumission
    • Voir documentation de la balise <button> (ouverture dans un nouvel onglet)

      Pensez bien à changer la valeur de l'attribut "type" du bouton en "submit". Sinon, vous ne pourrez pas soumettre votre formulaire.

    • Voir documentation de la balise <input> (ouverture dans un nouvel onglet)
  3. Soumettez le formulaire et observez l'url de la page dans votre navigateur

Partie 3 (cliquer pour afficher / cacher)

  1. Rajouter une balise <textarea> avec le label associé et les bons attributs
    • Voir documentation <textarea> (ouverture dans un nouvel onglet)
  2. Rajouter des inputs type radio (min. 2)
    • Voir documentation des boutons radio (ouverture dans un nouvel onglet)
  3. Rajouter des inputs type checkbox
    • Voir documentation des cases à cocher (ouverture dans un nouvel onglet)
  4. Soumettez le formulaire et observez l'url de la page dans votre navigateur

    Note : Si l'url devient trop longue, n'hésitez pas à la copier et la coller dans VS Code pour l'observer plus facilement.

Partie 4 (cliquer pour afficher / cacher)

  1. Rajouter une liste déroulante (balise <select>)
    • Voir documentation de la liste déroulante (ouverture dans un nouvel onglet)
    • Outil pour générer une liste rapidement (ouverture dans un nouvel onglet)

Partie 5 (cliquer pour afficher / cacher)

  1. Rajoutez l'attribut "required" sur l'un des champs ajouté précédemment (plusieurs si vous le souhaitez)
    • Voir documentation de l'attribut "required" (ouverture dans un nouvel onglet)
  2. Soumettez le formulaire sans remplir les champs que vous avez défini comme requis

Pour aller plus loin (cliquer pour afficher / cacher)

  • Appliquez ce que vous venez d’apprendre en ajoutant de nouveaux champs à votre formulaire
