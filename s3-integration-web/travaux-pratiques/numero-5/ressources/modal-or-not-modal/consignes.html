Modales et popover — Consignes TP

Les consignes ci-dessous sont à adapter sur le site inclus avec la consigne. Site qui a été réalisé avec tailwindcss. Vous pouvez donc utiliser l'ensemble des classes proposés par l'outil pour produire un design de qualité rapidement.

Partie 1 (cliquer pour afficher / cacher)
  1. Au chargement de la page, affichez une balise <dialog> avec, au minimum, le contenu suivant :
    • Le nom et le numéro du Pokémon
    • Ses types
    • Son sprite
    • Un bouton pour fermer la balise <dialog>

    • Voir documentation sur la balise <dialog>

    La mise en page du <dialog> est à votre discrétion.

  2. Affichez une balise <dialog> avec les mêmes informations que précédemment, mais cette fois-ci en cliquant sur un des Pokémon (contenus chacun dans une balise <button>). La balise doit apparaitre en mode "modale" (avec un arrière-plan)
    Etant donné que vous découvrez le javascript, ce n'est pas grave si vous affichez la même information sur chaque Pokémon cliqué.
    Pour ce faire, vous aurez besoin des documentations suivantes — ouverture dans un nouvel onglet :
    • addEventListener() pour gérer l'évènement du clic
    • Les méthodes .showModal() et .close()
    • querySelectorAll() pour sélectionner tous les boutons

      Il faudra combiner querySelectorAll() avec une boucle (for ou forEach) pour appliquer l'évènement de clic sur chaque élément

    Pour rappel, voici la syntaxe d'une fonction : 
    const myFunction = (/* argument1, argument2... */) => {
    /* instructions */
}
  3. Jouez sur le pseudo-element ::backdrop en changeant son apparence. Testez les trois versions suivantes :
    • Changez la couleur
    • Affichez une image
    • Flouttez l'arrière-plan

      Pour flouter l'arrière-plan, il faudra utiliser la propriété backdrop-filter (ouvrir documentation dans un nouvel onglet)

    • Voir documentation du pseudo-élément ::backdrop

Partie 2 (cliquer pour afficher / cacher)
  1. Affichez la modale affichée précédemment, cette fois-ci sous forme de popover
    • Voir documentation de l'attribut popover
  2. Ajoutez un bouton dans la popover pour la fermer

Partie 3 (cliquer pour afficher / cacher)
  1. Effectuez une transition lors de l'ouverture et la fermeture d'un <dialog> ou d'un popover
    • Voir documentation transition <dialog>
    • Voir exemples de transition sur popover et <dialog>

Pour aller plus loin (cliquer pour afficher / cacher)
  1. En vous aidant du data-attribute "data-pokemon-data", affichez de façon dynamique le contenu du popover/<dialog>. Le sprite doit également être affiché

    Les données sont encodées sous forme de chaîne de caractères qui répond au standard du format JSON. Pour transformer, cette chaîne de caractères en vrai JSON, il faut utiliser la fonction JSON.parse() — ouvrir documentation dans un nouvel onglet

    Le JSON est de la forme suivante : 

    {"id":1,"nom":"Bulbizarre", "types":["plante", "poison"]}
    • Voir documentation des data-attributes
  2. Indiquer le Pokémon qui a été sélectionné
