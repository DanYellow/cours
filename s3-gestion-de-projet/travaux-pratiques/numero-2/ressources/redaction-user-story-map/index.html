Contexte de l'exercice (cliquer pour afficher / cacher)

On a vu précédemment que les Stories (et leurs tests d'acceptation) sont très utiles. Toutefois, elles se placent dans un ensemble qui peut être représenté par une User Story Map. Il permet d'organiser les Stories selon une chronologie qui permet d'avoir une vue d'ensemble du produit. De ce fait, selon le produit, cette Story Map peut être très dense aussi bien verticalement qu'horizontalement.

  • Guide - Comment découper une Story (ouverture dans un nouvel onglet) (Vous avez également le document en local si jamais le lien ne fonctionne pas dans le TP#1)

Consignes de la pratique

Partie 1 (cliquer pour afficher / cacher)

En groupe, réalisez la "morning routine" médiane du groupe pour vous préparer pour aller à l'IUT (du réveil jusqu'au départ de chez vous) sous forme de User Story Map. Nous pouvons déjà dégager au minimum trois gros thèmes/features avec leur(s) épique(s) :

  • Chambre
    • Réveil
  • Alimentation
    • Petit-déjeuner
    • Déjeuner
  • Départ

Chaque membre groupe ayant sa propre routine matinale, il n'est pas utile de perdre du temps à faire une routine par membre et fusionner le tout ensuite. Dès le début, faites la routine la plus représentative du groupe.

Vous pouvez ajouter d'autres thèmes / features, si vous en avez besoin.

Vous devez organiser, sur l'axe vertical, en fonction des priorités, pas l'ordre logique des actions, les tâches (User Stories).

A vous rajouter ceux restants ainsi que les epics et user stories. Il n'est pas utile de respecter le gabarit "en tant que..." pour les user stories, un titre, sous forme de verbe, suffit, bien évidemment ça doit représenter une tâche atomique.

Pour réaliser votre Story Map, vous pouvez utiliser un outil collaboratif comme Figma ou les post-it fournis. Essayez d'avoir un code couleur entre vos thèmes/features, epics et user stories

Pensez à lister "toutes" les tâches : ouvrir les volets, mettre les chaussons, éteindre le réveil...

Partie 2 (cliquer pour afficher / cacher)

A partir d'un des thèmes suivants, réalisez son user story map. Pensez bien à ordonner vos features / epics dans l'ordre chronologique, et les stories par priorité. Il n'est pas utile de respecter le gabarit "en tant que..." pour les user stories, un titre suffit, bien évidemment ça doit représenter une tâche atomique.

  • Répondre à un message sur un forum - L'utilisateur n'est pas connecté à la base
  • Réservation d'un voyage (moyen de transport et hébergement inclus) sur un site web
  • Achat d'un produit sur un site d'e-commerce jusqu'au paiement pour un internaute ayant déjà un compte sur le site
