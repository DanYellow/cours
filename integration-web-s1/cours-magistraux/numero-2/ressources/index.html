Cours Magistral S1 - Référencement Naturel / Search Engine Optimization

Le SEO (Search Engine Optimization) ou Référencement Naturel représente une part importante dans la construction d'un site web à quoi bon de développer le meilleur site du monde si à la fin personne ne le trouve.

Dans cette ressource, vous avez deux versions du même site, si l'apparence est la même (et le contenu plus ou moins), la structure du code HTML est, quant à elle, différente. Les balises sont mieux choisies ainsi que les attributs, permettant ainsi d'obtenir un meilleur score par Lighthouse. On l'a vu en cours, le code HTML représente une grande part du référencement naturel d'un site web, un code de mauvaise qualité peut très fortement sanctionner un site web.

Testez le score Lighthouse des deux sites. Attention tout de même, Lighthouse ne fonctionne qu'avec le protocole HTTP(S), il faut donc que le site tourne sur un serveur. De même pour Screamingfrog. Pensez à déplacer le contenu des sites dans WAMP/MAMP ou un serveur distant.

Au-delà de l'audit sur Screamingfrog ou Lighthouse, pensez bien à comparer les deux codes pour voir les différences.

  • Exemple bon SEO - Ouverture dans un nouvel onglet

Les images du "bon-exemple" utilisent le format avif, ce format d'image, très bien géré de nos jours, possède l'avatange de proposer une compression sans perte. De plus, il permet même la gestion de la transparence, comme pour le format png. Ainsi, si vous souhaitez diminuer drastiquement la taille de vos images, pensez au format avif. Il existe des convertisseurs en ligne pour vos images, comme le site squoosh.app - ouverture dans un nouvel onglet.

Pensez au format avif, votre site se chargera plus vite et vos ferez faire des économies de bande passante à vos utilisateurs.

Ces deux exemples utilisent des technologies / notions que nous aurons l'occasion de voir plus tard dans le cursus. Mais si ceci vous intéresse rien ne vous empêche de vous informer sur tailwindcss (abordé en S2) ou encore le responsive web design (abordé en S3).
