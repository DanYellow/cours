Consignes de la pratique - importmap

Partie 1 (cliquer pour afficher / cacher)
  1. Créez un fichier html
  2. Cherchez un ou plusieurs modules sur le site jsdelivr.com - ouverture dans un nouvel onglet ou skypack.dev - ouverture dans un nouvel onglet Définissez un importmap et importez-le dans voter fichier html.

    L'import de modules ne fonctionne qu'avec... les modules. Assurez-vous que le script soit ESM (EcmaScript Module). Si ce n'est pas le cas, le script ne sera pas chargé et le navigateur lèvera une erreur.

    • Voir documentation d'importmap - Ouveture dans un nouvel onglet
  3. Utilisez le(s) module(s) importé(s) dans votre script.

    N'oubliez pas de marquer votre script comme étant de type module dans la balise <script>, sinon ça ne fonctionnera pas.

Partie 2 (cliquer pour afficher / cacher)

L'importmap fonctionne également avec des modules locaux, il est possible de définir un module en local pour ensuite le définir dans l'importmap. Le fonctionnement reste le même. Quand le module est local, vous pouvez utiliser un chemin relatif ou absolu, en sachant que c'est l'importmap qui charge les modules et non le fichier HTML. Les chemins doivent donc être faits entre les modules et l'importmap.

  1. Créez un module local et ajoutez-le à votre importmap

    Ce module peut contenir une variable ou une fonction qui fait juste des addition. Intuile de faire quelque chose d'élaboré.

  2. Utilisez ce module fraîchement crée dans votre script

Partie 3 (cliquer pour afficher / cacher)

    Il faut sélectionner le fichier csv de la partie "Fichiers par département de naissance".

  2. Consultez le notebook "initiation-pandas-2.ipynb"
