Partie 2 (cliquer pour afficher / cacher)

Il est également possible en javascript de définir des fonctions. Pour rappel, une fonction permet de définir un ensemble d'instructions réutilisables. Il est possible de définir des valeurs d'entrées, appelées paramètres, qui peuvent être modifiées à l'envie lors de l'appel de la fonction en passant des arguments. 

const example = (nom, prenom) => {
    return `Bonjour ${prenom} ${nom}`;
}

example("Helena", "Despoux");

En javascript, il existe deux syntaxes pour écrire une fonction, une syntaxe dite "moderne" (appelée aussi ES2015) et l'autre "ancienne". Les deux sont valables, il y a quelques différences entre les deux. Libre à vous d'utiliser l'une ou l'autre, toutefois, dans le cadre du cours, nous utiliserons la syntaxe moderne.

Pour que vous puissez observer la différence, voici deux extraits de code qui font la même chose mais avec deux syntaxes différentes.

Le code dans les deux onglets font la même chose. 

  • const maPremiereFonction = (param1, param2) => {
    return param1 + param2;
}

maPremiereFonction(45, 20);

  • Cette syntaxe est l'ancienne façon de définir des fonctions en javascript, il est possible que vous la voyez encore en ligne. Elle est et sera toujours gérée par les navigateurs. 

    function maPremiereFonction(param1, param2) {
    return param1 + param2;
}
maPremiereFonction(45, 20);

Dans le code ci-dessus, on définit une fonction nommée "maPremiereFonction" prenant deux paramètres et retourne leur somme. Puis on appelle notre fonction avec comme arguments les nombres 45 et 20. Ceci veut donc dire que, dans la fonction "maPremiereFonction", les paramètres "param1" et "param2" ont pour valeur 45 et 20 respectivement. C'est très simple mais c'est très bien pour commencer. A noter que les opérateurs mathématiques de base sont les mêmes qu'en PHP :

  • + : addition
  • - : soustraction
  • * : multiplication
  • / : division

Pour les commentaires, pour rappel, il vous suffit de préfixer la ligne de code avec // ou utiliser le raccourci Ctrl / Cmd + ⇧ Maj (macOS) + / en plaçant le curseur sur une ligne ou ensemble de lignes sur VS Code.

Comme le HTML et le CSS, les commentaires en javascript sont accessibles à l'utilisateur final. Par conséquent : ne mettez pas de données sensibles dedans.

  1. Réutilisez la fonction ci-dessus et affichez le résultat grâce à la méthode console.log()
    • Vous pouvez stocker le retour de la fonction dans une variable ou appeler directement la fonction dans la méthode console.log()
    • Voir documentation de la méthode console.log() (ouverture dans un nouvel onglet)
    • Voir documentation des fonctions (ouverture dans un nouvel onglet)
  2. Créez les fonctions suivantes :
    • Prend en paramètre un nombre et le multiplie par lui-même
    • Prend en paramètres deux chaînes de caractères et souhaite bonjour à cette personne

      Pour ce faire, il faudra concaténer une chaîne de caractères.

      • Voir documentation sur la contacténation de chaînes de caractères (ouverture dans un nouvel onglet)
    • Prend en paramètres deux nombres, les multiplie et retourne toujours un produit strictement positif
    • Prend en paramètres deux nombres qui les multiplie sans utiliser l'opérateur de multiplication

Ces fonctions doivent impérativement retourner un résultat (mot-clé"return") et doivent voir leur résultat être affiché dans un console.log().

Exemple :

// Définition de la fonction
const maFonction = () => {
    return "hello";
}
// Appel de la fonction. N'oubliez pas les parenthèses, 
// indispensables pour appeler la fonction
console.log( maFonction() );

N'oubliez pas de nommer vos fonctions avec des noms clairs et explicites tout comme les paramètres de vos fonctions. A noter deux choses :

  • Une fonction ne peut pas commencer par un nombre (comme les variables)
  • Une fonction peut contenir des lettres accentuées, mais évitez de le faire

Si votre code ne se comporte pas de la façon attendue. N'oubliez pas d'utiliser la console du navigateur, onglet "Console". Le navigateur vous indiquera le type d'erreur ET la ligne où se trouve l'erreur, vous pouvez même cliquer dessus pour l'afficher.

Partie 3 (cliquer pour afficher / cacher)

Créez les fonctions suivantes :

  1. Prend en paramètre une chaîne de caractères et retourne "Bonjour " + le nom du département dispensé à l'IUT de Sarcelles. Si le département n'est pas proposé à l'IUT de Sarcelles, la fonction doit retourner "Bonjour mais ce département n'est pas proposé à l'IUT de Sarcelles"
  2. Prend en paramètre un nombre entier et qui retourne une chaîne de caractères (ces chaînes sont là à titre indicatif, vous pouvez adapter en fonction de vos préférences) :

    • "Vous êtes une personne agée" si la valeur n'est dans aucune des bornes suscitées ou n'est pas une valeur correcte
  3. Retourne true, si une chaîne de caractères est en minuscules dans sa totalité

Ces fonctions doivent impérativement retourner un résultat (mot-clé"return") et doivent voir leur résultat être affiché dans un console.log().

    const cible = document.querySelector("selecteur");

cible.addEventListener('click', () => {
    console.log(maFonction());

  2. Créez plusieurs balises <p> avec du texte et la même classe CSS. Puis, grâce à la propriété "textContent", faites changer le contenu textuel de ces balises lorsqu'on clique sur un bouton (vous pouvez réutiliser le même ou un créer un nouveau)

    // On récupère tous les éléments ayant la classe "ma-classe"
const listeElementsMaClasse = document.querySelectorAll(".ma-classe");

// On parcourt notre liste d'éléments
for (let i = 0; i < listeElementsMaClasse.length; i++) {
    // listeElementsMaClasse[i] représente un élément ayant la classe "ma-classe"
    listeElementsMaClasse[i].addEventListener('click', maFonction)
}

Partie 5 (cliquer pour afficher / cacher)
  1. Toujours au clic sur le même bouton, ajoutez une classe CSS sur un seul élément. Il faudra utiliser un data-attribute sur l'élément que l'on souhaite cibler.

    Pour cibler par data-attribute, il faut mettre le nom du data-attribute entre crochets. Exemple :

    // Contient le premier élément qui a le data-attribute "data-attribut-personnalise"
document.querySelector("[data-attribut-personnalise]")

    A noter que le ciblage par "crochets" permet cibler n'importe quel élément par la valeur de son attribut HTML.

    Le code ci-dessus cible donc le premier élément trouvé ayant le data-attribute "data-attribut-personnalise". A noter que cette syntaxe est aussi valable dans un fichier CSS.

    Si vous souhaitez cibler un data-attribut avec une valeur spécifique, il faut procéder de la façon suivante : 

    // Contient le premier élément ayant le data-attribute 
// "data-attribut-personnalise" avec la valeur "ma-valeur"
document.querySelector("[data-attribut-personnalise='ma valeur']");

    Cette classe CSS, à ajouter/retirer, devra être rajoutée dans un fichier CSS. Pensez à mettre des propriétés CSS dans la classe pour voir des modifications ainsi que lier votre fichier html à votre fichier CSS.

    • Voir documentation la propriété classList (ouverture dans un nouvel onglet)
    • Voir documentation des data-attributes (ouverture dans un nouvel onglet)

Partie 6 (cliquer pour afficher / cacher)
  1. Ajoutez un champ de texte de type "text" (pas besoin de le mettre dans un formulaire)
  2. Ciblez le champ via un data-attribute et ajoutez l'évènement "input". Cet évènement permet d'écouter tout changement sur un champ "en temps réel"
  3. Affichez le contenu de la balise <input /> dans une balise <p> qui aura été elle-même ciblée avec un data-attribute

    Pour récupérer le contenu textuel de notre <input> (ou <textarea>), il faudra récupérer l'élément ciblé par notre fonction addEventListener contenu dans l'objet "evt". Par exemple :

    cible.addEventListener('input', (evt) => {
    // texteChamp contient le texte actuellement dans le champ de texte
    const texteChamp = evt.target.value;
})
