Partie 1 (cliquer pour afficher / cacher)
  1. Proposez une version responsive du site contenu dans le dossier site/. La mise en page est actuellement pensée pour mobile (portrait et paysage). Adaptez-la pour proposer une expérience convenable sur tablette et ordinateur de bureau/portable. Vous pouvez ajouter des balises html si cela est nécessaire.
    • Voir documentation des modifiers responsive de tailwindcss (ouverture dans un nouvel onglet)
    • Voir documentation des modifiers d'orientation d'écran de tailwindcss (ouverture dans un nouvel onglet)
    • Voir documentation des plages de breakpoints de tailwindcss (ouverture dans un nouvel onglet)

N'oubliez pas le mode responsive pour tester les différentes tailles et orientation d'écran. C'est plus pratique que réduire la taille de la fenêtre.
